# DISCLAIMER

## Purpose of this repository

This repository is a **local-first prototype** for probabilistic regional seismic risk modeling in Mexico.

It is intended to demonstrate:

- bounded prototype implementation
- disciplined data handling
- reproducible analytical workflows
- baseline model evaluation
- professional engineering documentation

It should be read as a technical and research-oriented prototype, not as an operational safety system.

---

## What this repository is

This repository is:

- a bounded prototype
- a local-first implementation
- an open-source-only workflow
- a portfolio-grade engineering artifact
- an exploratory probabilistic analysis system

---

## What this repository is not

This repository is **not**:

- an official earthquake warning system
- a deterministic earthquake prediction system
- an operational emergency response platform
- a production deployment
- an institutional or government safety service
- a substitute for seismological authorities
- a guarantee of future seismic events

---

## Interpretation boundary

Any outputs generated by this repository must be interpreted as:

- exploratory probabilistic analytical results
- prototype artifacts derived from bounded data and baseline methods
- evidence for architectural and methodological discussion

They must **not** be interpreted as:

- deterministic forecasts
- public alert decisions
- civil protection recommendations
- emergency instructions

---

## Scientific boundary

Earthquake occurrence is governed by complex tectonic and geophysical processes.

This repository does not model the full physical dynamics of the Earth system and does not claim predictive certainty.

Any signal detected by the prototype should be understood as a bounded statistical observation subject to severe uncertainty, data limitations, and methodological constraints.

---

## Operational boundary

This repository is intentionally designed as a **local-first, low-cost prototype**.

Current implementation choices, including SQLite as prototype authority and lightweight API/web surfaces, are appropriate for a bounded engineering demonstration.

They are not evidence of production readiness.

---

## Data boundary

Raw source data should be treated as immutable evidence.

Curated records, analytical features, model outputs, and visualizations are derived artifacts generated for prototype use.

Users are responsible for verifying source provenance, date ranges, and limitations before drawing conclusions from the outputs.
